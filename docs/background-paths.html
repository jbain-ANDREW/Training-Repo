Background reading — optional Paths cause more beginner frustration than almost anything else. This page explains what they are, why Windows is particularly tricky, and what the mysterious PATH environment variable actually does. Knowing this makes error messages much less cryptic.

Your File System Is a Tree

Every file on your computer lives inside a folder, which lives inside another folder, all the way up to a root. On Windows, the root is a drive letter like C:\. On Mac and Linux, the root is just /.

C:\
├── Users\
│ └── yourname\
│ ├── Documents\
│ │ └── Github\
│ │ └── my-project\
│ │ └── index.html
│ └── Downloads\
└── Program Files\

A path is the sequence of folder names you follow from the root to reach a specific file. It's an address — just like a street address tells you how to get to a building, a path tells the computer how to get to a file.

Reading a Path

Here's the full path to the index.html file from the tree above:

C:\Users\yourname\Documents\Github\my-project\index.html
C: Drive letter folder Folder names index.html File name \ Separator (Windows)

Each \ is a separator — it means "go inside this folder." Read left to right: start at drive C, enter Users, enter yourname, enter Documents, and so on.

Absolute vs. Relative Paths

Absolute path

Starts from the root and works anywhere, regardless of where you currently are. It's the full address. Always unambiguous.

# Absolute — works from anywhere
C:\Users\yourname\Documents\Github\my-project\index.html

Relative path

Describes a location relative to where you currently are. If you're already inside my-project\, you can just say index.html. If you need to go up a level first, use ...

# You are currently in: C:\Users\yourname\Documents\Github\my-project\
# Relative path to same file — just the filename
index.html

# Relative path to a file one folder up
..\other-project\readme.txt
In terminal commands, relative paths are shorter and often more convenient. Absolute paths are safer when you're not sure what your current directory is.

Windows vs. Mac/Linux: The Separator Problem

Windows uses a backslash (\) as the folder separator. Mac and Linux use a forward slash (/). This tiny difference causes real problems when:

  • You copy a path from a Windows tutorial into a Mac terminal (or vice versa)
  • A Python script uses hardcoded Windows paths but runs on a server (Linux)
  • VS Code's terminal (which uses PowerShell or bash) behaves differently from cmd
Python's os.path module and the newer pathlib library handle this automatically — they give you the right separator for whatever OS you're on. Avoid writing \ or / in Python file paths directly.

Also: Windows paths are case-insensitive (Documents and documents mean the same thing). Mac and Linux are case-sensitive. Code that works on Windows can silently fail when moved to a Linux server.

The PATH Environment Variable

When you type a command in the terminal — like python, git, or code — the OS needs to find the actual program file to run. It doesn't search your entire computer. Instead, it searches a specific list of folders called the PATH.

PATH is an environment variable: a named piece of text that the OS makes available to all programs. It contains a list of folder paths separated by semicolons.

# See your current PATH (PowerShell)
PS> $env:PATH
C:\Windows\System32;C:\Program Files\Git\cmd;C:\Users\you\AppData\Local\Programs\Python\Python312\Scripts;...

When you type python, the OS walks down this list of folders and stops at the first one that contains a file named python.exe. The order matters — whichever folder comes first wins.

Type: python
→
Search PATH folders
in order
→
Found! Run python.exe
in first matching folder

PATH controls which program runs when you type a command name.

Why this causes problems with Python

If you install Python multiple times (or conda creates environments), there can be multiple python.exe files in different folders. The one at the top of PATH wins — which may not be the one you intended. This is why the Python installer has a critical checkbox: "Add Python to PATH". If you skip it, typing python in the terminal does nothing.

Checking "Add Python to PATH" during installation puts the new Python folder at the front of PATH, making it the default. If you have an older Python that you want to keep as default, be careful with this option.

Common Windows Path Problems

Spaces in folder names

C:\Users\John Smith\Documents — that space in "John Smith" will break many terminal commands unless you put the whole path in quotes:

# This fails if the path has spaces
cd C:\Users\John Smith\Documents

# This works
cd "C:\Users\John Smith\Documents"

The AppData folder is hidden

Many tools (including Python, pip, and VS Code extensions) install things into C:\Users\yourname\AppData\. This folder is hidden by default in File Explorer. You can see it in the terminal or by typing the path directly into File Explorer's address bar.

"Command not found" errors

If you install a tool and the terminal says it can't find it, the tool's folder probably isn't in PATH. The fix: either add it to PATH manually (search "Edit the system environment variables" in Windows) or reinstall the tool and look for a "Add to PATH" option during setup.
