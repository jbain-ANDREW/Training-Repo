What Is GitHub Desktop?

GitHub Desktop is a free app that gives you a visual interface for all the Git operations that the terminal exercise does with commands. Every concept — clone, branch, commit, push, pull request — is the same. You just click instead of type.

If you haven't done the terminal exercise yet, you can start here — the concepts will still make sense. But if you do both, you'll understand Git much more deeply. The callout at the bottom explains why.

Step 1 — Install GitHub Desktop

  1. Download GitHub Desktop

    Go to desktop.github.com and click the download button for Windows.

    Screenshot: desktop.github.com download page [ screenshot: desktop.github.com — Windows download button ]
  2. Run the installer

    The installer runs silently and opens GitHub Desktop when done — there are no options to configure.

  3. Sign in to GitHub

    GitHub Desktop will ask you to sign in. Click Sign in to GitHub.com and use your GitHub account credentials.

    Screenshot: GitHub Desktop sign-in screen [ screenshot: GitHub Desktop — "Sign in to GitHub.com" button ]

Step 2 — Clone the Repository

  1. Open the Clone dialog

    In GitHub Desktop, go to File → Clone repository…

    Screenshot: GitHub Desktop — File menu → Clone repository [ screenshot: GitHub Desktop File menu — Clone repository ]
  2. Find the Training Repo

    Click the URL tab and paste:

    https://github.com/jbain-ANDREW/Training-Repo

    Choose a local folder where you want the project saved, then click Clone.

    Screenshot: GitHub Desktop clone dialog with URL filled in [ screenshot: Clone dialog — URL tab — Training-Repo URL pasted — Clone button ]
What just happened? GitHub Desktop downloaded the entire repository to your machine, exactly like git clone does in the terminal.

Step 3 — Create Your Branch

  1. Click "Current Branch"

    At the top of GitHub Desktop you'll see a bar showing the current branch (main). Click it.

    Screenshot: GitHub Desktop — Current Branch dropdown showing "main" [ screenshot: GitHub Desktop top bar — "Current Branch: main" dropdown ]
  2. Create a new branch

    Click New Branch, type your name in lowercase with a hyphen (e.g. jane-smith), and click Create Branch.

    Screenshot: GitHub Desktop — New Branch dialog with name typed [ screenshot: New Branch dialog — "jane-smith" typed — Create Branch button ]
What just happened? You created a private branch, the same as git checkout -b firstname-lastname in the terminal.

Step 4 — Make Your Splash Page

  1. Open the repo folder in VS Code

    In GitHub Desktop, click Open in Visual Studio Code (or go to Repository → Open in Visual Studio Code).

    Screenshot: GitHub Desktop — "Open in Visual Studio Code" button [ screenshot: GitHub Desktop — "Open in Visual Studio Code" button ]
  2. Copy the template

    In VS Code's file explorer (left panel), navigate to exercises/splash-page/. Right-click template.html and choose Copy. Then navigate to docs/splash-pages/, right-click and choose Paste. Rename the pasted file to firstname-lastname.html.

    Screenshot: VS Code file explorer — right-click copy/paste of template file [ screenshot: VS Code file explorer — copy template.html → paste into docs/splash-pages/ → rename ]
  3. Edit your file

    Open your renamed file and customize the title, background color, emoji, and personal info. The template has numbered comments guiding each change.

Step 5 — Commit Your Changes

  1. Switch back to GitHub Desktop

    GitHub Desktop will automatically detect your new file and show it in the Changes panel on the left.

    Screenshot: GitHub Desktop — Changes panel showing new file [ screenshot: GitHub Desktop Changes panel — new file listed with a green + icon ]
  2. Write a commit message

    At the bottom-left, type a short description in the Summary box, e.g. Add splash page: Firstname Lastname.

  3. Click "Commit to firstname-lastname"

    The button at the bottom confirms the branch name. Click it.

    Screenshot: GitHub Desktop — commit message filled in, Commit button [ screenshot: GitHub Desktop — Summary: "Add splash page: Jane Smith" — Commit to jane-smith button ]
What just happened? You saved a snapshot of your changes — the same as git add + git commit in the terminal, but in one click.

Step 6 — Push and Open a Pull Request

  1. Push your branch to GitHub

    Click the Publish branch button at the top of GitHub Desktop (it may say Push origin if you've pushed before).

    Screenshot: GitHub Desktop — "Publish branch" button in top bar [ screenshot: GitHub Desktop top bar — "Publish branch" button ]
  2. Create a Pull Request

    After pushing, GitHub Desktop shows a Create Pull Request button. Click it — your browser will open to the GitHub PR form with your branch pre-selected.

    Screenshot: GitHub Desktop — "Create Pull Request" button [ screenshot: GitHub Desktop — "Create Pull Request" button ]
  3. Submit the Pull Request on GitHub

    Add a title, fill in the checklist, and click Create pull request. Your instructor will review and merge it, and your page will appear in the Splash Page Archive.

Only edit files inside docs/splash-pages/. Changes outside that folder will not be accepted.

GitHub Desktop vs. the Terminal — What's the Same?

Terminal command GitHub Desktop equivalent
git clone <url> File → Clone repository → paste URL
git checkout -b branch-name Current Branch → New Branch → type name
git add <file> Check the checkbox next to the file in Changes
git commit -m "message" Type message in Summary → click Commit
git push origin branch-name Publish branch / Push origin button
Open PR on github.com Create Pull Request button → browser opens

Now try it the other way

You just completed the full GitHub workflow by clicking through GitHub Desktop. Now try the same exercise using terminal commands — no buttons, just text. The operations are identical; the experience is very different. Engineers use both depending on the task, and knowing both makes you more adaptable.
