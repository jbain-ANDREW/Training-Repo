How Feedback Works

This site uses GitHub Issues to track problems and suggestions. Submitting an issue is a one-click process — and since you already have a GitHub account for this course, there's nothing extra to sign up for.

Reporting a problem here is also a small extra taste of how real software teams work — filing issues is how bugs and improvements get tracked in professional projects.

What Kind of Problem?

🔴

Something is broken

A link doesn't work, a page won't load, a step in a guide is wrong
🟡

Something is confusing

A step is unclear, terminology isn't explained, I got lost
🔵

Something is missing

A topic I expected isn't covered, a setup scenario isn't addressed
🟢

I have a suggestion

An idea for improvement, a better way to explain something

How to File an Issue

  1. Click one of the cards above

    It will open a pre-filled GitHub Issue form for this repo.

  2. Fill in the title and description

    Be as specific as you can — which page, which step, what you expected vs. what happened.

  3. Click "Submit new issue"

    That's it. Your report is logged and visible to the instructor.

Or if you prefer, go directly to the issues page and file one from scratch:

Open a GitHub Issue →

Good Issue vs. Vague Issue

A good issue makes it easy to find and fix the problem. Here's the difference:

Vague Specific
"The guide doesn't work" "On the Fresh Install page, Step 3: when I run python --version I get 'not recognized' even after checking the PATH box"
"Step 2 is confusing" "Python Cleanup page, Situation B — the instructions say to open 'App Execution Aliases' but I can't find it in Windows 11 search"
"Add more stuff" "The complexity page doesn't cover what to do when VS Code shows the wrong interpreter — that tripped me up"
