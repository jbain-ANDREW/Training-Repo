Background reading — optional Once you have Claude or Copilot set up, you'll discover there are two very different ways to work with it. Understanding the difference — and when to use each — is one of the most useful things you can learn.

Mode 1: Copy-Paste

This is the most obvious way to use an AI coding tool. You open a browser tab with Claude or ChatGPT, type a question or paste in some code, read the response, and manually copy whatever looks useful into your editor.

It works. For a quick question — "what does this error mean?" or "write me a function that does X" — it's perfectly fine. But it has real friction built in.

Copy-paste workflow
Write code in editor
→
Copy code to clipboard
→
Paste into browser chat
→
Describe the problem
→
Read AI response
→
Copy code from browser
→
Paste into editor
→
Test it
→
Repeat for each error

The red steps are pure friction — moving text between two places. When you're iterating quickly on something, those context switches add up fast.

Mode 2: Cowork (Agentic)

In cowork mode, the AI tool runs inside your editor and has direct access to your files. You describe what you want in the chat panel; the AI reads your code, makes the changes, and applies them directly. You never leave the editor.

This is what the Claude and Copilot VS Code extensions enable. Instead of copying and pasting, you're having a conversation with something that can see your entire project and act on it.

Cowork workflow
AI reads your file(s)
→
You describe what you want
→
AI makes the change directly
→
You review and accept
→
Continue the conversation

Copy-paste

  • Works from any browser
  • No setup beyond an account
  • Good for one-off questions
  • AI has no context about your project
  • You do all the copy-paste work
  • Errors require another round-trip

Cowork / Agentic

  • AI sees your open files automatically
  • Changes applied in-place — no pasting
  • Multi-step tasks possible in one conversation
  • Error messages fed back to AI instantly
  • Much faster iteration loop
  • Requires VS Code + extension setup

Why Context Makes Such a Difference

When you use copy-paste mode and paste a snippet of code, you're giving the AI a fragment without surrounding context — other files, imports, variable names defined elsewhere, the specific error message it just produced. The AI works with what you give it.

In cowork mode, the AI can see the full file, or multiple files, or even browse your project structure. A question like "why isn't this function finding my data?" becomes much easier to answer when the AI can also see the file that defines where the data lives.

Think of the difference between calling someone on the phone to describe your car problem versus having a mechanic who can see the car, run diagnostics, and make the repair right there. Both work. One is dramatically more efficient.

Good Prompts in Both Modes

Regardless of which mode you use, the quality of your prompt determines the quality of the response. Vague prompts produce vague code. Specific prompts produce useful code.

Weak prompt Fix my code.
Better prompt My Python script reads a CSV file and should print the average of the "score" column. It runs without errors but prints 0.0 instead of the correct value. The CSV file has a header row. Here's the script: [paste code]

In cowork mode, you can be even more direct because the AI already has the context:

Cowork prompt (AI has the file open) The calculate_average function is returning 0.0 instead of the correct average. Find the bug and fix it.
You don't need to know how to fix the bug yourself to write a good prompt. You do need to be specific about what the code is supposed to do and what it's actually doing instead. The AI handles the "how" — you handle the "what."

How This Course Uses Both Modes

The exercises in this course are designed to work in either mode. The splash page exercises give you copy-paste prompts that you can paste into Claude on the web — that's intentionally simple so you can try it immediately without setup.

Once you have Claude or Copilot running inside VS Code, the same exercises become faster and more iterative. You can ask follow-up questions, ask it to explain what it changed, and modify the result without leaving your editor.
