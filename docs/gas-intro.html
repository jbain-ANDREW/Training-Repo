What Is Google Apps Script?

Google Apps Script (GAS) is a scripting platform built into Google's cloud. You write JavaScript in a browser-based editor at script.google.com — no terminal, no install, no environment to configure. Your code runs on Google's servers.

🌐
Runs in the browser

Write code at script.google.com from any computer. Nothing to install.

📦
Google services built in

Access Sheets, Docs, Gmail, Drive, Forms, and Calendar with a few lines of code.

🚀
Deploy as a web app

Get a public URL that runs your script — like a mini web server, free.

🤖
Great for AI generation

Claude and Copilot know GAS well. Describe what you want and paste the result.

Where GAS shines: automating repetitive tasks (filling a Sheet, sending emails), building lightweight web apps (forms, dashboards, link pages), and connecting Google services together — all without a server or local environment.

Example: Personal Link Page

This is the complete code for a personal link page (think mini Linktree) deployed as a Google Apps Script web app. When someone visits your URL, doGet() runs and returns an HTML page. Paste it into a new project and you'll have a live URL in under five minutes.

Code.gs

function doGet() {
  return HtmlService.createHtmlOutput(buildPage())
    .setTitle('My Links')
    .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);
}

function buildPage() {
  const name    = 'Your Name';
  const tagline = 'Student — Spring 2026';
  const avatar  = '🚀';

  const links = [
    { label: 'GitHub',    url: 'https://github.com/your-username' },
    { label: 'LinkedIn',  url: 'https://linkedin.com/in/your-profile' },
    { label: 'Portfolio', url: 'https://your-site.github.io' },
    { label: 'Email Me',  url: 'mailto:you@example.com' },
  ];

  const linkHTML = links.map(link =>
    `<a href="${link.url}" target="_blank">${link.label}</a>`
  ).join('');

  return `<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <style>
    * { box-sizing: border-box; margin: 0; padding: 0; }
    body {
      font-family: 'Segoe UI', sans-serif;
      background: #1a1a2e;
      color: white;
      min-height: 100vh;
      display: flex;
      flex-direction: column;
      align-items: center;
      justify-content: center;
      padding: 2rem;
      text-align: center;
    }
    .avatar  { font-size: 4rem; margin-bottom: 0.8rem; }
    h1       { font-size: 2rem; margin-bottom: 0.3rem; }
    .tagline { opacity: 0.65; margin-bottom: 2rem; }
    .links   { display: flex; flex-direction: column; gap: 0.75rem; width: 100%; max-width: 320px; }
    .links a {
      display: block;
      background: rgba(255,255,255,0.1);
      color: white;
      text-decoration: none;
      padding: 0.75rem 1rem;
      border-radius: 8px;
      font-size: 1rem;
      transition: background 0.2s;
    }
    .links a:hover { background: rgba(255,255,255,0.2); }
  </style>
</head>
<body>
  <div class="avatar">${avatar}</div>
  <h1>${name}</h1>
  <p class="tagline">${tagline}</p>
  <div class="links">${linkHTML}</div>
</body>
</html>`;
}
How it works: doGet() is a reserved function name — Google calls it automatically when someone visits your web app URL. It returns an HtmlOutput object built from a JavaScript template string. The links array maps to <a> tags using .map() and .join().

Deploy Your Link Page

Follow these steps to get a live URL. You only need a Google account.

  1. Open the editor

    Go to script.google.com and click New project in the top-left corner.

    Screenshot placeholder script.google.com — click "New project"
  2. Name your project

    Click "Untitled project" at the top and rename it to something like My Link Page.

  3. Replace the default code

    Delete the starter myFunction() and paste the full Code.gs above. Update name, tagline, avatar, and the links array with your own information.

    Screenshot placeholder Apps Script editor with Code.gs pasted in
  4. Save the project

    Press Ctrl+S (or the save icon). The title bar should show the project name without an asterisk when saved.

  5. Deploy as a web app

    Click Deploy → New deployment. Set type to Web app. Under "Who has access" choose Anyone. Click Deploy.

    Screenshot placeholder Deploy dialog — type: Web app, access: Anyone
  6. Copy your URL and test it

    After deploying, Google gives you a URL that looks like https://script.google.com/macros/s/AKf.../exec. Open it in a new tab — you should see your link page.

    Screenshot placeholder Deployment complete screen showing the web app URL
Making changes after deploying: edits to the code don't automatically update the live URL. Go to Deploy → Manage deployments, click the pencil icon, and choose New version to push updates.

AI Generation Exercises

These exercises use Claude or Copilot to extend the link page. Each one gives you a starting prompt — paste it into Claude at claude.ai or into Copilot Chat in VS Code. Copy the generated code back into your Apps Script editor and redeploy.

Exercise 1
Add a visit counter Beginner

Google Apps Script has a built-in key-value store called Properties Service that persists between page loads. Ask the AI to add a counter that tracks how many times your page has been visited and displays it at the bottom.

Prompt for Claude or Copilot I have a Google Apps Script web app with a doGet() function that returns an HTML link page. Add a visit counter using PropertiesService.getScriptProperties() that increments each time doGet() is called and displays the total visit count at the bottom of the page in small muted text. Keep all existing HTML and styling intact.

What to look for: The AI should add getProperty / setProperty calls in doGet() before the return, and inject the count into the HTML template string.

Exercise 2
Add a typewriter tagline Intermediate

The HTML returned by HtmlService can include <script> tags with JavaScript that runs in the visitor's browser. Ask the AI to replace the static tagline with a typewriter effect that cycles through several phrases.

Prompt for Claude or Copilot I have a Google Apps Script web app that returns an HTML link page. The page has a <p class="tagline"> element with a static subtitle. Replace it with a typewriter effect that cycles through these phrases: "Student — Spring 2026", "Python & JavaScript", "Ask me about Google Apps Script", "Building things with AI". Use a pure JavaScript setTimeout loop. The script tag should be inside the returned HTML string.

What to look for: The AI should embed a <script> block in the template string with the same typewriter pattern from the splash page exercises.

Exercise 3
Pull data from a Google Sheet Intermediate

One of GAS's superpowers is reading directly from a Google Sheet without any API keys or authentication setup. Ask the AI to move your links list into a Sheet so you can update them without touching code.

Prompt for Claude or Copilot I have a Google Apps Script web app. I want to store my links in a Google Sheet instead of hardcoding them. The Sheet has two columns: column A is the link label, column B is the URL. Modify doGet() to read all rows from the Sheet using SpreadsheetApp.openById(SHEET_ID).getActiveSheet().getDataRange().getValues(), filter out empty rows, and build the links array from that data. Replace SHEET_ID with a placeholder comment. Keep the rest of the HTML unchanged.

What to look for: The AI should add a SpreadsheetApp.openById() call, map the 2D array from getValues() into the links format, and note where you paste your Sheet ID.

Exercise 4
Build a different tool entirely Challenge

Now that you know GAS can serve HTML, read from Sheets, and persist data — build something new from scratch using the AI. Some ideas: a simple RSVP form that writes responses to a Sheet, a countdown timer to an event date, or a public status page for a project.

Example prompt (adapt it) Write a complete Google Apps Script web app with a doGet() function that returns a single-page RSVP form. The form should collect name, email, and a yes/no attendance field. Submitting the form should call a doPost() function that appends the data to a Google Sheet (use a placeholder SHEET_ID). After submission, show a thank-you message. Style the page with a dark background (#1a1a2e), white text, and centered layout. No external libraries or frameworks — plain HTML, CSS, and JavaScript only.

What to look for: The AI may split logic across doGet() and doPost(). Make sure the form's action attribute is set to the script URL and the method is POST, or the AI uses google.script.run for client-side calls.

Good to Know

Execution limits

Free Google accounts get 6 minutes of execution time per script run and a daily quota of 90 minutes total. For the kinds of scripts on this page — serving HTML, reading a Sheet — you will never come close to those limits.

Sharing and permissions

When you deploy with "Anyone" access, anyone with the URL can view the page without signing in. If your script reads a Sheet, that Sheet's data becomes visible too — don't put anything private in a publicly deployed app.

Re-deploying after changes

Saving the code does not update the live URL. You must create a new version: Deploy → Manage deployments → edit pencil → New version → Deploy.

Using AI for debugging: if your GAS code throws an error, copy the full error message from the Apps Script execution log (View → Executions) and paste it into Claude or Copilot along with your code. The error messages are specific enough that the AI can usually identify the fix immediately.
