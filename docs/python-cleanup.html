Why This Gets Complicated

Windows makes it easy to end up with multiple Python installations without realizing it. Common culprits:

  • Python installed from python.org (the right way)
  • Python installed from the Microsoft Store (causes PATH conflicts)
  • Anaconda or Miniconda — bundles its own Python
  • A previous course that installed an older version

Having multiple versions is not necessarily a problem, but you need to know which one your terminal is actually using.

Step 1 — Find Out What You Have

  1. Open a terminal

    Press Win + R, type cmd, press Enter.

  2. Check which Python your terminal finds first 
    python --version

    This shows the version that runs when you type python. Note the version number.

    Screenshot: cmd — python --version output [ screenshot: cmd — python --version → e.g. Python 3.9.7 ]
  3. Find where it lives 
    where python

    This lists every Python executable Windows can find, in order of priority. The first line is the one that runs when you type python.

    Screenshot: cmd — "where python" listing multiple paths [ screenshot: where python — may show multiple paths like C:\Users\...\AppData\Local\Programs\Python\... ]

Reading the output of where python

Path contains… What it means Status
AppData\Local\Programs\Python Installed from python.org — the clean option Good
AppData\Local\Microsoft\WindowsApps Microsoft Store Python — causes issues Problematic
anaconda3 or miniconda3 Conda Python — fine if you use Conda intentionally Context-dependent
Program Files\Python System-wide install from python.org Fine

Step 2 — Common Situations & Fixes

Situation A: "python is not recognized"

Python is installed but not on your PATH. The quickest fix:

  1. Re-run the python.org installer

    Download the latest installer from python.org, run it, and this time choose Modify. On the next screen, look for "Add Python to environment variables" and check it.

    Screenshot: Installer Modify screen — "Add Python to environment variables" checkbox [ screenshot: installer Modify → Optional Features → "Add Python to environment variables" ]
  2. Close and reopen your terminal

    PATH changes don't apply to already-open terminals. Close cmd and open it again, then try python --version.

Situation B: Microsoft Store Python is first in PATH

The Microsoft Store version of Python has known issues with file paths and permissions on Windows. It is best to disable it.
  1. Open App Execution Aliases

    Press Win, search for "Manage app execution aliases", open it.

    Screenshot: Windows search — "Manage app execution aliases" [ screenshot: Windows search showing "Manage app execution aliases" ]
  2. Turn off the Python aliases

    Find python.exe and python3.exe in the list and toggle them Off.

    Screenshot: App execution aliases — python.exe toggled Off [ screenshot: python.exe and python3.exe toggles set to Off ]
  3. Reopen your terminal and verify 
    where python
python --version

    The Microsoft Store path should no longer appear.

Situation C: Old version from a previous course

Having an old version alongside a new one is not a problem as long as you know which one is active. If you want to clean it up:
  1. Open Apps & Features

    Press Win, search "Add or remove programs", open it. Search for Python.

    Screenshot: Add or remove programs — Python entries listed [ screenshot: Apps & features — multiple Python versions listed ]
  2. Uninstall the old version(s)

    Click the old version, click Uninstall. Leave the most recent python.org version in place.

Situation D: Anaconda installed and you want to keep it

Anaconda installs its own Python and takes over the PATH. If you want the standard python.org Python to be your default:

  1. Do not uninstall Anaconda — it may be needed for other courses. Leave it in place.
  2. Use the Anaconda Prompt for Conda work — it sets PATH correctly for Conda. Use the regular cmd or VS Code terminal for standard Python work.
  3. In VS Code, make sure the selected interpreter is the python.org one, not the Conda one (check the bottom-left status bar when a .py file is open).
    Screenshot: VS Code status bar — Python interpreter selector [ screenshot: VS Code bottom bar — Python 3.12.x interpreter selected ]

Step 3 — Confirm a Clean State

After any cleanup, run these three checks. All three should succeed:

python --version
pip --version
where python
The path shown by where python should point to AppData\Local\Programs\Python or Program Files\Python — not to WindowsApps, Anaconda, or anything unexpected.

Still Stuck?

If none of the above solves your problem, report it with the output of where python and python --version included in your description.
