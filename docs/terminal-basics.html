What Is a Terminal?

A terminal (also called a command line, shell, or console) is a text-based way to control your computer. Instead of clicking icons, you type commands and the computer responds with text.

It might look intimidating at first, but you only need a small handful of commands to do everything in this course. By the end of this page you will know all of them.

The terminal is not fragile — typing a wrong command doesn't break anything. The worst that usually happens is an error message. Experiment freely.

Opening a Terminal on Windows

Windows has several terminal options. Here are the three you'll encounter:

Option 1 — VS Code Terminal (recommended)

If you have VS Code open, press Ctrl+` (the backtick key, top-left of the keyboard). A terminal panel will open at the bottom of VS Code. This is the most convenient option because it's already in the right folder for your project.

Screenshot: VS Code with terminal panel open at the bottom save as: assets/images/terminal-vscode.png [ screenshot: VS Code — terminal panel open at bottom, showing a prompt ]

Option 2 — Command Prompt (cmd)

Press Win + R, type cmd, press Enter.

Screenshot: Windows Run dialog with "cmd" typed save as: assets/images/terminal-run-cmd.png [ screenshot: Win+R dialog — type "cmd" → Enter ]

Option 3 — PowerShell

Press Win, search for PowerShell, click to open. PowerShell is more powerful than cmd and is the default terminal in newer versions of Windows. For this course, cmd and PowerShell are interchangeable.

VS Code's terminal starts in your project folder automatically. cmd and PowerShell start in your home folder (C:\Users\yourname). You'll need to navigate to your project folder manually — see the cd command below.

Reading the Prompt

When the terminal is ready for input, it shows a prompt — a line ending with > or $. This is where you type commands.

# This is what a prompt looks like
C:\Users\yourname> _

The text before > shows your current location in the file system (your working directory). This matters — commands like git clone create files in the folder you're currently in.

The Commands You Need

You only need five commands for this course. Here they are:

Command What it does Example
pwd Print Working Directory — shows where you are right now pwd
ls List — shows files and folders in the current directory
(use dir in cmd if ls doesn't work)		 ls
cd Change Directory — move to a different folder cd Documents
cd .. (go up one level)
mkdir Make Directory — create a new folder mkdir my-project
cp Copy — copy a file to a new location cp file.html copy.html
In VS Code's terminal on Windows, both ls and dir work. In plain cmd, only dir works. When in doubt, use dir.

Try It — Navigate Your File System

  1. Open a terminal

    Use any of the three options above. You should see a prompt.

  2. Find out where you are

    Type pwd and press Enter. You'll see a path like C:\Users\yourname.

    C:\Users\yourname> pwd
    C:\Users\yourname
  3. See what's in the current folder 
    ls

    You'll see a list of files and folders. These are the same things you'd see in File Explorer.

  4. Move into a folder

    Pick any folder from the list and navigate into it:

    cd Documents

    Notice the prompt changes to show your new location.

  5. Go back up one level 
    cd ..

    .. always means "the folder above this one". Two dots, no space.

Useful Shortcuts

Tab completion

Start typing a folder or file name and press Tab — the terminal will complete it for you. If there are multiple matches, press Tab again to cycle through them. This saves a lot of typing and prevents typos.

Command history

Press the ↑ arrow key to cycle through commands you've previously run. No need to retype long commands.

Clearing the screen

Type cls (cmd) or clear (PowerShell / VS Code terminal) to clear the terminal output. Your work is not deleted — the screen just gets cleaned up.

Cancelling a running command

Press Ctrl+C to stop whatever is currently running and return to the prompt.

Common Mistakes

"The system cannot find the path specified" or "No such file or directory"

Usually means you're in the wrong folder. Run pwd to see where you are, and ls to see what's available. Check your spelling — file paths are case-sensitive on some systems.

Nothing happens after I type a command

Make sure you pressed Enter. Commands only run when you press Enter.

The terminal seems stuck

A running command (like a long install) can look like nothing is happening. Wait a moment. If it's genuinely stuck, press Ctrl+C to cancel.
